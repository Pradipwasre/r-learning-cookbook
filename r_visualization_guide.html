R Data Visualization

A Complete Guide to Creating Beautiful Plots

Introduction to Plots

In this guide, we will explore three fundamental types of plots that are essential for data visualization:

Scatter Plot

Shows the relationship between two continuous variables

Histogram

Displays the distribution of a single variable

Box Plot

Summarizes data distribution using quartiles

Your First Plot

# Create two vectors a <- c(10,13,14,10,12,17) b <- c(15,17,19,13,13,19) # Create a simple scatter plot plot(a, b)

How This Works:

c() function creates a vector (a list of numbers)

plot(a, b) creates a scatter plot where 'a' values are on x-axis and 'b' values are on y-axis

Each point on the graph represents a pair: (a[1], b[1]), (a[2], b[2]), etc.

Working with Real Data: NYC Flights

# Install and load the package (run once) # install.packages("nycflights13") library(nycflights13) # Attach the dataset for easy access attach(nycflights13::flights) # Plot arrival delay vs departure delay plot(arr_delay, dep_delay)

Understanding the Code:

library() loads the package into your R session

attach() makes all column names directly accessible without using dataset$column

arr_delay and dep_delay are columns from the flights dataset

Customization: Adding Titles and Labels

Step 1: Adding a Main Title

plot(arr_delay, dep_delay, main="Arrival vs Departure Delay")
Parameter Purpose Effect on Graph
main Sets the main title of the plot Displays text at the top center of the graph

Step 2: Adding Axis Labels

plot(arr_delay, dep_delay, main="Arrival vs Departure Delay", xlab="Arrival Delay", ylab="Departure Delay")
Parameter Purpose Effect on Graph
xlab Label for x-axis Shows text below the x-axis to describe horizontal data
ylab Label for y-axis Shows text beside the y-axis to describe vertical data

Adding Reference Lines

plot(arr_delay, dep_delay, main="Arrival vs Departure Delay", xlab="Arrival Delay", ylab="Departure Delay") # Add a diagonal line (slope=1) abline(0, 1) # Add a horizontal line at y=0 abline(0, 0)

Understanding abline():

abline(a, b) draws a line with equation: y = a + bx

abline(0, 1) creates y = 0 + 1x, a 45-degree diagonal line

abline(0, 0) creates y = 0, a horizontal line through origin

Why it matters: Reference lines help identify patterns. The diagonal line shows where arrival and departure delays are equal.

Changing Plot Characters (pch)

# Using different point shapes plot(arr_delay, dep_delay, main="Arrival vs Departure Delay", xlab="Arrival Delay", ylab="Departure Delay", pch=3) # Using letters as points plot(arr_delay, dep_delay, main="Arrival vs Departure Delay", xlab="Arrival Delay", ylab="Departure Delay", pch="s")
pch Value Shape When to Use
1 Circle Default, general purpose
2 Triangle Highlighting specific categories
3 Plus (+) When you have many overlapping points
4 Cross (×) Similar to plus, alternative style
5 Diamond Special or important points
6 Reverse Triangle Contrasting with regular triangles
7 Box and Cross Complex datasets needing distinction
"s" Letter 's' Any letter or character can be used

Why pch Matters:

Different point shapes help distinguish between multiple data series on the same plot, making patterns easier to identify visually.

Filtering and Layering Data

# Plot only United Airlines data plot(arr_delay[carrier=="UA"], dep_delay[carrier=="UA"]) # Add American Airlines data on top with different shape points(arr_delay[carrier=="AA"], dep_delay[carrier=="AA"], pch=3)

Understanding Filtering:

[carrier=="UA"] is a filter that selects only rows where carrier equals "UA"

points() adds new points to an existing plot without erasing it

Why this works: You build up complex visualizations layer by layer

Adding Colors

# United Airlines in red plot(arr_delay[carrier=="UA"], dep_delay[carrier=="UA"], col="red") # American Airlines in green points(arr_delay[carrier=="AA"], dep_delay[carrier=="AA"], col="green")
Parameter Purpose Effect on Graph
col Sets the color of points Changes point color to help distinguish different data groups

Multi-Panel Plots

# Create a 1 row × 2 column layout par(mfrow=c(1, 2)) # First plot plot(arr_delay[carrier=="UA"], dep_delay[carrier=="UA"], main="UA") # Second plot plot(arr_delay[carrier=="AA"], dep_delay[carrier=="AA"], main="AA")

Understanding par(mfrow):

par() sets graphical parameters that affect all subsequent plots

mfrow=c(1, 2) creates a grid with 1 row and 2 columns

c(2, 2) would create 2 rows and 2 columns (4 plots total)

Why use this: Compare multiple plots side-by-side for better analysis

Equal Scales for Better Comparison

par(mfrow=c(1, 2)) plot(arr_delay[carrier=="UA"], dep_delay[carrier=="UA"], main="UA", xlim=c(0, 1000), ylim=c(0, 1000)) plot(arr_delay[carrier=="AA"], dep_delay[carrier=="AA"], main="AA", xlim=c(0, 1000), ylim=c(0, 1000))
Parameter Purpose Effect on Graph
xlim Sets x-axis limits Forces x-axis to span from minimum to maximum value specified
ylim Sets y-axis limits Forces y-axis to span from minimum to maximum value specified
Pro Tip: Always use the same xlim and ylim values when comparing multiple plots. This ensures you are comparing apples to apples.

Resetting to Single Plot

# Return to single plot layout par(mfrow=c(1, 1)) plot(arr_delay[carrier=="UA"], dep_delay[carrier=="UA"], main="UA")
